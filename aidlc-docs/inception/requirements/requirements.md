# 테이블오더 서비스 - 요구사항 문서

## Intent Analysis

- **User Request**: 디지털 테이블오더 서비스 구축 (고객 주문 + 관리자 모니터링)
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI, 관리자 UI, 서버, 데이터베이스)
- **Complexity**: Moderate (단일 매장, 소규모, MVP 중심)

---

## 1. 기술 스택 결정사항

| 항목 | 선택 |
|---|---|
| 프레임워크 | TypeScript + Next.js (풀스택) |
| 데이터베이스 | SQLite (Prisma ORM 활용) |
| 실시간 통신 | Server-Sent Events (SSE) |
| 인증 | JWT 토큰 기반 |
| 배포 환경 | 로컬 개발 환경 (Docker Compose) |
| 이미지 관리 | 미포함 (텍스트만, MVP) |

---

## 2. 비즈니스 컨텍스트

- **단일 매장** 운영 (멀티테넌시 불필요)
- **소규모**: 1개 매장, 테이블 20개 이하
- **관리자 계정**: 시드 데이터로 1개 생성 (회원가입 기능 없음)
- **보안 확장 규칙**: 미적용 (MVP/프로토타입 수준)

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer Interface)

#### FR-C01: 테이블 태블릿 자동 로그인
- 관리자가 초기 설정 시 매장 식별자, 테이블 번호, 비밀번호 입력
- 로그인 정보 로컬 저장 후 자동 로그인
- **세션 만료 없음** (관리자 수동 리셋 방식)

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명 (이미지 미포함)
- 카테고리 간 빠른 이동
- 터치 친화적 UI (최소 44x44px 버튼)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공 시: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 테이블 정보, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 이용 완료 처리된 주문 제외

### 3.2 관리자용 기능 (Admin Interface)

#### FR-A01: 매장 인증
- 사용자명 + 비밀번호 로그인 (시드 데이터 계정)
- JWT 토큰 기반, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- **테이블 이용 완료**: 확인 팝업 → 세션 종료 → 주문 내역 과거 이력 이동 → 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리 (이미지 URL 미포함)
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 통신: 주문 발생 후 2초 이내 관리자 화면 표시
- 소규모 환경 (20테이블 이하) 기준 최적화

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 터치 타겟)
- 직관적인 카드 형태 레이아웃
- 명확한 시각적 계층 구조

### NFR-03: 데이터 무결성
- 주문 생성 시 트랜잭션 처리
- 장바구니 로컬 저장 (새로고침 시 유지)
- 세션 기반 주문 그룹화

## 6. 데이터 모델 개요

### 핵심 엔티티
- **Admin**: 관리자 계정 (시드 데이터)
- **Table**: 테이블 정보 (번호, 비밀번호, 현재 세션 ID)
- **Category**: 메뉴 카테고리
- **MenuItem**: 메뉴 항목 (메뉴명, 가격, 설명, 카테고리, 노출 순서)
- **Order**: 주문 (테이블, 세션 ID, 상태, 총 금액, 주문 시각)
- **OrderItem**: 주문 항목 (메뉴명, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (이용 완료 시 이동)

### 테이블 세션 라이프사이클
1. 첫 주문 시 새 세션 ID 자동 생성
2. 세션 동안 주문 누적
3. 관리자가 "이용 완료" 처리 시 세션 종료
4. 주문 내역 → OrderHistory로 이동
5. 테이블 현재 주문/총액 리셋
6. 다음 고객의 첫 주문 시 새 세션 시작
